# OpenPrimer: Project Guidelines

## 🌟 Vision
OpenPrimer is the first universal, free, and open-source knowledge manual, synthesizing the entirety of academic knowledge (from primary school to Bachelor's degree) via artificial intelligence. It positions itself as a **transition tool** between classical learning (books/lectures) and the future of education (personalized AI tutoring).

## 🎨 Design & UX Principles
- **Discretion & Elegance**: The interface (inspired by Gemini/ChatGPT) must fade away to make room for knowledge.
- **Dual-Pane Experience**: 
    - **Left (Reference)**: The static, structured, and academic manual.
    - **Right (Companion)**: The contextual AI Tutor for dialogue and explanation.
- **Atomic Knowledge**: Digestible course segments (5-10 min) locked by immediate validations.

## 🧠 Technological Strategy
- **MDX Pipeline**: Knowledge is stored in Markdown + React to allow total interactivity (Quizzes, Simulations, Diagrams).
- **Vector Intelligence**: Navigation by concepts via a semantic "Knowledge Map".
- **Auto-Wiki**: A reporting system allowing a "Correction" AI to continuously revise courses based on user feedback.
- **Multi-modality**: Integration of oral (audio) and simulation for deep validation of learning.

## 🚀 POC Goals (Proof of Concept)
Demonstrate feasibility on two pillars:
1. **Biology L1** (Scientific expertise)
2. **Computer Science L1** (Technical expertise)
*Each module must include the tutor, glossary, quizzes, and progression dashboard.*

## 📜 License & Governance
OpenPrimer is a common good. All generated content is under an open license. Revision is hybrid (AI-First with final human validation via an Admin console).
